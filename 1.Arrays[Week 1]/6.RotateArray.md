
```java
class Solution {
    public void rotate(int[] nums, int k) {
        int n=nums.length;
        k=k%n;
        reverse(nums,0,n-1);
        reverse(nums,0,k-1);
        reverse(nums,k,n-1);
    }
    public void reverse(int[] nums, int start,int end){
        while(start<end){
            int temp=nums[start];
            nums[start]=nums[end];
            nums[end]=temp;
            start++;
            end--;
        }
    }
}
```
Complexity
Time: O(n)
Space: O(1)

This is the standard interview solution
Dry Run

Example:

nums = [1,2,3,4,5,6,7]
k = 3
Step 1: Reverse entire array
7 6 5 4 3 2 1
Step 2: Reverse first k elements
5 6 7 4 3 2 1
Step 3: Reverse remaining elements
5 6 7 1 2 3 4

Answer ✔

Interview Questions
Q1. Why take k = k % n?

If

n = 7
k = 10

Rotating by 10 is the same as rotating by:

10 % 7 = 3

This avoids unnecessary rotations.

Q2. Why reverse three times?

The three reversals reposition the elements without using extra space:

Reverse the whole array.
Reverse the first k elements.
Reverse the remaining n-k elements.

This achieves the rotation in-place.

Other approaches
1.Brute force
Rotate the array by one position k times 
Time:O(n × k) 
Space:O(1)
2.Create new array
more space 
create temp array store current value temp[(i+k)%n]=nums[i] and then copy back
Time:O(n)
Space:O(n)
Which one should you mention in an interview?

If asked, you can present them in this order:

Brute force — simplest but inefficient.
Extra array — improves time to O(n) using extra space.
Reverse algorithm — optimal O(n) time and O(1) space; the most commonly expected solution.
Cyclic replacement — also optimal, but more complex to implement and explain.
