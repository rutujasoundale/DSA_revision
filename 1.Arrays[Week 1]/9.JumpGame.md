Brute Force solution (based on dp) 
1.Read the problem
2.Try to draw tree based on possiblities to reach the last index
3.return true when reached or fase when unable 
```java
class Solution {
    public boolean canJump(int[] nums) {
        return solve(nums,0);
   }
    public boolean solve(int[] nums,int index){
//return true when reached last index
        if(index>=nums.length-1){
            return true;
        }
//try every possible jump
        for(int jump=1;jump<=nums[index];jump++){
            if(solve(nums,index+jump)){
                return true;
            }
        }
        return false;
    }
}
```
Dry Run

Input:

nums = [2,3,1,1,4]

Call:

solve(0)

jump = 1 → solve(1)

    jump = 1 → solve(2)

        jump = 1 → solve(3)

            jump = 1 → solve(4)

                reached end
                return true

Since one path succeeds, every previous call returns true.

Worst Case

Consider:

[4,4,4,4,4,4,4]

At every index, you branch into many recursive calls.

Approximate recursion tree:

          0
      / / | \
     1 2 3 4
    /|\...

This explores an exponential number of paths.

Time Complexity: O(2^n) (or more generally exponential)
Space Complexity: O(n) (maximum recursion depth)
Why is it inefficient?

The same subproblem is solved repeatedly.

For example, solve(3) might be reached from:

0 → 1 → 3
0 → 2 → 3
0 → 1 → 2 → 3

This overlapping work is exactly why we optimize the brute-force solution with memoization (DP) and eventually arrive at the O(n) greedy solution.

Optimal Greedy approach
```java
class Solution {
    public boolean canJump(int[] nums) {
        //lets try maintaing var farthest
        int farthest=0;
        for(int i=0;i<nums.length;i++){
        if(i>farthest){
            return false;
        }
        farthest=Math.max(farthest,i+nums[i]);
        }
        return true;
    }
}
```
Complexity
Time: O(n) (one pass)
Space: O(1)
Interview Explanation (30 seconds)

"I maintain the farthest index that can be reached while scanning the array. If I ever encounter an index greater than this farthest reachable position, it means there's no way to get there, so I return false. Otherwise, I update the farthest reachable index as max(farthest, i + nums[i]). If I finish the traversal, the last index is reachable. This greedy approach runs in O(n) time with O(1) extra space."
