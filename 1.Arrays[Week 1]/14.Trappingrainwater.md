The **Trapping Rain Water** problem is all about **finding how much water each bar can hold**.

### Step 1: Think like rain

Imagine it rains on buildings.

```
Height = [3, 0, 2, 0, 4]

      █
█     █
█   █ █
█   █ █
█ █ █ █
---------
```

Where will water stay?

Water gets trapped in the gaps between taller buildings.

---

## Step 2: What decides water at one position?

Take this position:

```
3  0  2  0  4
   ^
```

Can water rise above height 3?

No, because the left wall is only 3.

Can it rise above height 4?

Yes, right wall is 4.

The **shorter wall limits the water**.

So,

```
Water level = min(left wall, right wall)
```

Here,

```
leftMax = 3
rightMax = 4

Water level = min(3,4)=3
```

Current height is 0.

```
Water trapped = 3 - 0 = 3
```

---

### Another position

```
3 0 2 0 4
    ^
```

Current height = 2

```
leftMax = 3
rightMax = 4

Water level = 3

Water = 3-2 =1
```

---

### Last gap

```
3 0 2 0 4
      ^
```

Current =0

```
leftMax =3
rightMax =4

Water =3-0=3
```

Total

```
3+1+3=7
```

# The Key Observation

For **every index**

```
water[i] =
min(max height on left,
    max height on right)
-
height[i]
```

or

```
water[i] = min(leftMax, rightMax) - height[i]
```

If negative,

```
0
```

because water cannot be negative.

---

# Why minimum?

Suppose

```
Left wall = 8
Right wall = 5
```

Water cannot reach 8.

It will spill from the shorter side.

```
████████
█      █
█      █
█      █
█      █
████████
```

Water stops at height **5**, not 8.

That's why we always use

```
min(leftMax, rightMax)
```

# How do we find leftMax and rightMax?

### Brute Force

For every index,

* scan left
* scan right

```
for every i
    find left max
    find right max
```

Time:

```
O(n²)
```

---

### Better

Store them.

```
leftMax[i]
```

contains maximum till i.

Example

```
Height

3 0 2 0 4

leftMax

3 3 3 3 4
```

Similarly,

```
rightMax

4 4 4 4 4
```

Now

```
water =
min(leftMax,rightMax)-height
```

Time

```
O(n)

Space

O(n)
```

# Optimal Two-Pointer Intuition (Most Important)

Instead of storing arrays, use two pointers.

```
l = 0
r = n-1
```

Maintain

```
leftMax
rightMax
```

### Why move the smaller side?

Suppose

```
leftMax = 3
rightMax = 7
```

The water on the **left pointer** depends only on `leftMax`.

Why?

Because the right side is already taller (7).

Even if it becomes 100 later,

```
min(3,100)=3
```

The answer doesn't change.

So we can safely calculate water on the left and move left.

Similarly,

If

```
rightMax < leftMax
```

then right determines the water, so process the right side.

---

## Rule

```
if(leftMax <= rightMax)
    process left
else
    process right
```

This is the core intuition behind the **O(n)**, **O(1)** solution.

---

# One-line intuition to remember

> **Water above a bar is limited by the shorter of the tallest bar on its left and the tallest bar on its right.**
> In the two-pointer approach, always process the side with the smaller maximum because that side's water level is already determined.

This intuition is the key to deriving the optimal two-pointer solution during interviews.



###1.Brute force
```java
class Solution {
    public int trap(int[] height) {
       int n=height.length;
       int watertrapped=0;
//Traverse left and  right to find leftmax and rightmax
       for(int i=0;i<n;i++){
        int leftmax=0;
        for(int j=0;j<=i;j++){
            leftmax=Math.max(leftmax,height[j]);
        }
        int rightmax=0;
        for(int j=i;j<n;j++){
            rightmax=Math.max(rightmax,height[j]);
        }
//Add [min from leftmax rightmax{Beacause shorter wall store water and max height it will be splitted} - currentposition
        watertrapped += Math.min(leftmax,rightmax)-height[i];
       } 
       return watertrapped;
    }
}
```
Time Complexity
O(n²)
Space Complexity
O(1)

###2.Better
1.Make two arrays left and right to store max values at each position
2.similarly compute water trapped by min logic
```java
class Solution {
    public int trap(int[] height) {
        int n = height.length;
        if (n == 0) return 0;

        int[] leftMax = new int[n];
        int[] rightMax = new int[n];

        leftMax[0] = height[0];
        for (int i = 1; i < n; i++) {
            leftMax[i] = Math.max(leftMax[i - 1], height[i]);
        }

        rightMax[n - 1] = height[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            rightMax[i] = Math.max(rightMax[i + 1], height[i]);
        }

        int water = 0;

        for (int i = 0; i < n; i++) {
            water += Math.min(leftMax[i], rightMax[i]) - height[i];
        }

        return water;
    }
}
```
Time Complexity
O(n)
Space Complexity
O(n)

###3.Most expected Optimal (Two Pointers) - O(n), O(1)

```java
class Solution {
    public int trap(int[] height) {
        int n=height.length;
        int left=0;
        int right=n-1;
        int water=0;

        int leftmax=0;
        int rightmax=0;

        while(left<=right){
            if(leftmax<=rightmax){
                if(leftmax<=height[left]){
                    leftmax=height[left];
                }
                else{
                    water+=leftmax - height[left];

                }
                left++;
            }
            else{
                if(height[right]>=rightmax){
                    rightmax=height[right];
                }
                else{
                    water+=rightmax - height[right];
                }
                right--;
                }
        }   
    return water;
    

    }
}
```
Time Complexity
O(n)
Space Complexity
O(1)
Interview Algorithm (Optimal)
```
Initialize:
left = 0
right = n - 1
leftMax = 0
rightMax = 0
water = 0

While left <= right:
    If leftMax <= rightMax:
        If height[left] >= leftMax:
            leftMax = height[left]
        Else:
            water += leftMax - height[left]
        left++
    Else:
        If height[right] >= rightMax:
            rightMax = height[right]
        Else:
            water += rightMax - height[right]
        right--

Return water
```

This two-pointer solution is the one most commonly expected in coding interviews because it achieves O(n) time with O(1) extra space.

# Why do we process the left side when `leftMax <= rightMax`?

Suppose:

```text
leftMax = 2
rightMax = 5
```

For the current left position:

```text
Water = min(leftMax, rightMax) - height[left]
      = min(2,5) - height[left]
      = 2 - height[left]
```

Even if the right side later becomes 100:

```text
min(2,100) = 2
```

The answer **doesn't change**. So the water at the left pointer is already determined, and it's safe to calculate it and move `left`.

The same reasoning applies symmetrically for the right side when `rightMax < leftMax`.

---

## Interview Tip

Whenever you're confused during the dry run, ask yourself:

> **Which side has the smaller maximum wall?**
> That side's trapped water is already fixed, so process that side first.

This single idea is the core intuition behind the optimal two-pointer solution.
