```java
class Solution {
    public int removeElement(int[] nums, int val) {
       int k=0;
        for(int i=0;i<nums.length;i++){
           if(nums[i]!=val){
            nums[k]=nums[i];
            k++;
           }
        }
        
        return k;
    }
}
```
Time Complexity
O(n) — each element is visited once.
Space Complexity
O(1) — no extra array is used.
Approach 1 (Brute Force)

Shift every element left whenever val is found.

Traverse once.

for every number

⭐ This is the expected interview solution.

Approach 4 (Swap with End)

Useful when the order doesn't matter.

Example

nums=[3,2,2,3]

Swap first 3 with last element.

3 2 2 3

↓

3 2 2 3

Decrease effective size.

Again

3

↓

2 2

Time

O(n)

Space

O(1)

This minimizes writes and is a good optimization when element order is irrelevant. LeetCode explicitly allows the order of the first k elements to change, so this is also a valid solution.
